## CRITICAL GAS RULES — NEVER VIOLATE
===========================================================

- Use VAR only — no const, let, arrow functions (=>),
  template literals (`${}`), for-of loops, or destructuring.
  Any of these will cause the file to silently fail to load.
- WASP write endpoints require ARRAY payload: [{...}]
  NOT object {…}
- clasp push REPLACES all remote GAS files — never add
  functions in the remote GAS editor only, or they will
  be deleted on next push.

## THE 5 SYNC FLOWS
===========================================================

F1: Katana PO received → WASP RECEIVING-DOCK
    - Trigger: Katana webhook (purchase_order.received)
    - Handler: handlePurchaseOrderReceived() in 02_Handlers.gs
    - Materials/Intermediates auto-route to PRODUCTION
    - Products go to RECEIVING-DOCK

F2: WASP qty add/remove/move → Katana Stock Adjustment
    - Trigger: WASP callout → doPost
    - Handler: handleWaspQuantityAdded/Removed/ItemMoved
      in 03_WaspCallouts.gs
    - 4-second batch window groups rapid callouts into 1 SA
    - F2_SKIP_LOCATIONS = ['SHOPIFY'] — prevents double-deduct
    - SYNC_CACHE_SECONDS = 120 prevents feedback loops
    - Logs ONLY to Adjustments tab (NOT Activity tab)

F3: Katana stock transfers → WASP (auto-poll)
    - Trigger: GAS time trigger (every N minutes)
    - Handler: 07_F3_AutoPoll.gs

F4: Katana MO complete → WASP deductions
    - Trigger: Katana webhook (manufacturing_order.done)
    - Handler: handleManufacturingOrderDone() in 02_Handlers.gs
    - Deducts ingredients, adds output with lot/batch

F5: ShipStation shipped → WASP SHOPIFY deduction
    - Trigger: ShipStation SHIP_NOTIFY webhook
    - Payload format: { resource_type: 'SHIP_NOTIFY',
                        resource_url: '...' }
    - Handler: handleShipNotify() in 06_ShipStation.gs
    - BUNDLE_MAP expands VB-* SKUs to component SKUs
    - Current bundles:
        VB-UFC-DUO:     UFC-4OZ x2, TTFC-1OZ x1
        VB-CPT-DUO-4OZ: B-PURPLE-GREEN-DUO x1, TTFC-1OZ x1
        VB-UHP:         UFC-4OZ x3, TTFC-1OZ x1, WH-8OZ-W x1
        VB-UFC-4OZ-SUB: UFC-4OZ x1

## KATANA TAB COLUMN LAYOUT (engin-src) — 15 COLUMNS
===========================================================

After 2026-02-27 fixes, the Katana tab has 15 columns:

  [0]  A  SKU
  [1]  B  Name
  [2]  C  Location (Katana location name)
  [3]  D  Type     (Katana category: PACKAGING/RAW MATER/etc.)
                   — display only, NOT used for routing
  [4]  E  Lot Tracked  (Yes/No)
  [5]  F  Batch        (lot number)
  [6]  G  Expiry       (date string)
  [7]  H  UOM
  [8]  I  K. Qty       (Katana stock qty)
  [9]  J  WASP Qty     (editable — user changes this to adjust)
  [10] K  Match        (MATCH/MISMATCH/ZERO/ONLY)
  [11] L  WASP Status  (Synced/Push/NEW/ERROR)
  [12] M  Orig WASP Qty (hidden, width=1 — stores baseline)
  [13] N  Adj Status   (Pending/ERROR/Pushed YYYY-MM-DD)
  [14] O  K.Type       (hidden, width=1 — Katana item type:
                        Material/Intermediate/Product)
                       — used by push engine for WASP routing

Push routing (pushKatanaAdjustments_ in 05_PushEngine.txt):
  itemType = row[14]  (K.Type hidden column)
  if katanaLoc === 'Storage Warehouse' → SW-STORAGE site
  if itemType === 'Material' || 'Intermediate' → PRODUCTION
  else → UNSORTED (default)

## LOGGING ARCHITECTURE (src/)
===========================================================

logToSheet(type, meta, detail)
  → Logger.log only, NOT written to sheet. Debug/trace.

logActivity(flow, detail, status, label, subItems)
  → Activity tab. Every F1/F3/F4/F5 operation.
  → F2 does NOT log here (by design, removed this session)

logFlowDetail(flow, execId, parent, children)
  → F1-F5 tabs. Per-flow audit trail.
  → F2 does NOT log here (removed this session)

logAdjustment(source, action, user, sku, itemName,
              site, location, lot, expiry, diff,
              katanaSaNum, saId, status)
  → Adjustments Log tab.
  → F2 (WASP callouts) logs here.
  → engin-src sheet pushes log here.

Color coding in Activity tab:
  qty red   (#cc0000) = deduction/removal
  qty green (#008000) = addition
  sub-item bg light red (#ffebee) = error
  sub-item bg light green (#e8f5e9) = success

## ROOT CAUSE OF -46002 ERRORS (for reference)
===========================================================

The -46002 "Insufficient quantity" error from WASP means
a REMOVE was attempted at a location that has no stock.

Root cause chain:
1. writeKatanaTab writes Katana category ("PACKAGING") to
   col D. Push engine read col D as itemType and checked
   itemType === 'Material' for PRODUCTION routing.
2. Category values never matched → ALL items routed to
   UNSORTED, even materials whose stock is at PRODUCTION.
3. REMOVE from UNSORTED fails → -46002.

Fixes applied:
  - Col D still shows category (user visibility)
  - Col O (row[14]) now has Katana item type for routing
  - Push engine reads row[14] for routing
  - -46002 retry also does full location search as fallback

## IMPORTANT PATTERNS
===========================================================

### Error code meanings (WASP):
  -46002  Insufficient quantity at location (REMOVE > stock)
  -57041  Lot/DateCode required (item is lot-tracked)
  -57009  Location does not exist
  -70010  Duplicate order number (treat as success)

### executeAction_() retry chain (05_PushEngine.txt):
  1. First attempt with smart-routed location
  2. -46002: search same location for real qty → retry
  3. -46002 + realQty=0: search ALL locations → retry
     from actual location (fix added 2026-02-27)
  4. -57041 ADD: retry with NO-LOT + future dateCode
  5. -57041 REMOVE: lookup actual lot from WASP → retry

### WASP advancedinventorysearch response structure:
  searchResult.Data.ResultList  (array of items)
  item.ItemNumber, item.LocationCode, item.SiteName
  item.QtyAvailable, item.Lot

### Katana API item types (from capitalizeType()):
  'material'     → 'Material'
  'intermediate' → 'Intermediate'
  'product'      → 'Product'
  Materials come from /materials API (always type='material')
  Products come from /products API (type from prod.type field)

### F2 feedback loop prevention:
  - SYNC_CACHE_SECONDS=120 in 00_Config.gs
  - F2_SKIP_LOCATIONS = ['SHOPIFY'] — WASP callouts from
    SHOPIFY location are ignored (F5 handles those)
  - Both use script cache keyed on SKU+action
